我想调整一下当前的邮箱以及账户存储方案：
1. 创建新账户时，尽量使用 .com 邮箱；
2. 如果aws账户创建失败，则不要缓存该邮箱账户；
3. 如果邮箱账户创建成功，则缓存该邮箱账户，并记录上次使用时间；
4. 使用[auto-register-aws-builder sessions --restore] 命令时，不需要执行检查过期账户和删除过期账户操作。只需要在给出账户列表后（需要显示上次使用时间，密码， 通过时间排序），用户选取时，通过 API 恢复该账户即可。显示新邮件时，需要显示全文内容。
